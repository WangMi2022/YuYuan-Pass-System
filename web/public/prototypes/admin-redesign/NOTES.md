# 后台重设计原型（评审后删除）

> 问题：资产管理中心的后台外壳与工作台应该采用哪种信息结构和视觉方向？
> 10 个方案在同一页面通过 `?variant=01..10` 切换，仅用于视觉决策，不接真实接口。

## 共同业务语境

- 用户：资产管理员、财务、IT 运维与部门负责人
- 页面任务：快速判断资产状态，并处理盘点、调拨、维修、领用等当日工作
- 固定样例数据：12,680 件实物、4,286 份档案、¥3,286 万原值、8 个告警、23 个待办
- 不变约束：中文后台、桌面高密度、移动端可读、键盘焦点清晰

## 10 个方向

| 编号 | 名称 | 主结构 | 字体策略 | 关键色 | 记忆点 |
| --- | --- | --- | --- | --- | --- |
| 01 | 秩序蓝图 | 左侧导航 + 技术图纸式工作区 | Bahnschrift / 微软雅黑 | `#235DFF` / `#0B1220` | 带坐标刻度的资产健康蓝图 |
| 02 | 资产指挥舱 | 图标轨道 + 实时态势面板 | DIN / Segoe UI | `#52E7A8` / `#07100E` | 运行状态带与告警雷达 |
| 03 | 财务台账 | 顶部导航 + 无卡片账簿 | Georgia / 微软雅黑 | `#067A56` / `#1D2A26` | 横向分栏账页与对账批注 |
| 04 | 生命周期轨道 | 顶部应用栏 + 中央环形流程 | Avenir / Segoe UI | `#6D5DFB` / `#19172C` | 入库到报废的环形状态轨道 |
| 05 | 仓库地图 | 区域导航 + 空间分布图 + 作业队列 | Arial Narrow / 微软雅黑 | `#1769E0` / `#F3B61F` | 以库区和楼层组织资产 |
| 06 | 今日行动队列 | 极窄导航 + 时间轴任务流 | Segoe UI / 微软雅黑 | `#E34B31` / `#231F20` | 以“现在该做什么”替代指标墙 |
| 07 | 数据分析画布 | 可折叠侧栏 + 全宽图表画布 | Inter / Bahnschrift | `#00A8E8` / `#101722` | 价值、分类、状态同屏联动 |
| 08 | 管理层简报 | 顶部导航 + 编辑式双栏报告 | Source Han Serif / 微软雅黑 | `#243C5A` / `#B9873D` | 像晨报一样呈现经营结论 |
| 09 | 模块矩阵 | 固定功能坞 + 高密度功能矩阵 | Arial / 微软雅黑 | `#5B5CE2` / `#16161A` | 业务模块本身就是可扫描仪表盘 |
| 10 | 命令中心 | 搜索/命令栏优先 + 结果工作区 | SF Mono / 微软雅黑 | `#2563EB` / `#111827` | 用一句命令直达资产和动作 |

## 运行与评审

```bash
cd web
npm run prototype:admin
```

- 点击底部编号，或按键盘左右方向键切换。
- URL 参数可直接分享，例如 `?variant=05`。
- 评审时分别记录：导航效率、信息密度、品牌气质、最想保留的局部。
- 选定方向后删除本目录，将胜出方向按正式工程规范重写到真实布局中。
